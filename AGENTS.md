# 무슨일 에이전트 운영 원칙

무슨일은 집회·시위 정보를 객관적으로 제공해 시민의 알권리를 돕는 서비스다. 동원, 여론전, 찬반투표, 불법/합법 단정 도구가 아니다.

## 비협상 원칙

1. 사용자 원문을 그대로 공개하지 않는다.
2. 자유 댓글, 추천/비추천, 찬반투표를 만들지 않는다.
3. 후원금은 노출, 랭킹, 신뢰도, 알림에 영향을 주지 않는다.
4. 신고 수만으로 자동 삭제하지 않는다.
5. 경찰, 정부, 언론, 주최 측, 시민, AI를 자동 진실로 처리하지 않는다.
6. 정확한 제보자 위치를 공개하지 않는다.
7. 얼굴, 차량번호, 주소, 전화번호, 미성년자 식별 정보는 공개 전 비식별 처리한다.
8. 집회 참여 독려, 충돌/회피/불법 진입/폭력/신상털이 안내를 만들지 않는다.
9. 모든 마스킹, 보류, 삭제, 정정, 반론, 병합, 분기, 상태 전환은 AuditLog 또는 TransparencyLog를 남긴다.

## 사용자 원문 직접 공개 금지

시민 제보, 신고, 반론, 정정의 원문은 저장할 수 있지만 공개 기본값이 아니다. 공개 화면에는 비식별 처리와 중립화된 Claim 요약만 노출한다.

금지 표현:

- 사용자가 쓴 문장을 그대로 카드/상세/알림에 노출.
- 실명, 전화번호, 주소, 차량번호, 얼굴, 미성년자 식별 정보 노출.
- 미확인 범죄 단정, 불법/합법 단정, 특정 개인 책임 단정.

## Claim 중심 모델

Event를 단일 진실 객체로 만들지 않는다.

- 모든 경찰/지자체 안내, 주최 측 공지, 언론 보도, 제보, 반론, 신고, AI 추정, 운영 기록은 Claim이다.
- 국회 의안과 현행 법령은 LawItem 공식 참조 메타데이터다. LawItem은 Event나 Claim이 아니며, IssueLawLink로만 Issue와 연결한다.
- Source provenance는 누가 말했는지다.
- Evidence strength는 무엇이 근거인지다.
- Risk level은 공개했을 때의 법적/안전 위험이다.
- 세 축을 합쳐서 "신뢰도" 하나로 뭉개지 않는다.

대상 모델:

- Issue
- Occurrence
- ContinuousPresence

공개 지도 모델:

- 자료 위치 핀: 공개 원천 자료에서 위치가 확인된 Occurrence 또는 ContinuousPresence만 표시한다.
- 현장 인증 영역: Proof-of-Presence를 통과하고 공개 가능한 현장 영상 Evidence 좌표로 계산한 흐림 Polygon만 표시한다.
- 교통선, 경로선, 검문/통제 지점, 인파 밀도 면은 공개 지도 도메인으로 만들지 않는다.

## Proof-of-Presence

사용자 제보는 실시간 GPS 기반 현장 영상만 허용한다. LIVE 제보는 다음 조건을 통과해야 한다.

- 앱 내 카메라 촬영.
- 촬영 후 제한 시간 내 업로드.
- foreground GPS.
- GPS 정확도 기준 충족.
- 대상 위치와 제보 위치 거리 기준 충족.
- Android Play Integrity 또는 iOS App Attest hook.
- 원본 해시 저장.
- 공개 전 EXIF 제거와 얼굴/차량번호/민감 OCR 마스킹.

갤러리 업로드, 캡처, 링크, 문서는 사용자 제보로 받지 않는다. 권리침해 신고와 반론은 제보가 아니라 별도 Claim 채널이다.
영상 Claim에 대한 현장 판단도 자유 투표가 아니라 Proof-of-Presence를 통과한 현장 인증 Claim으로만 저장한다.

## 조작 방어

국적이나 의견이 아니라 현장성으로 방어한다.

- 신규 계정 급증, 동일 문구 반복, 동일 기기/네트워크 군집, 위치 불일치, 비정상 이동속도, 자료 제보 급증, 대량 신고를 조작 신호로 본다.
- 삭제보다 지도/알림/우선순위 영향력 제거를 우선한다.
- 고위험 개인정보, 폭력 선동, 미성년자 노출 등은 즉시 비공개 검토 대상으로 보낸다.

## 위치/개인정보 보호

내부 정밀 좌표와 원본 미디어는 암호화 저장하고 접근 로그를 남긴다. 공개 좌표는 H3/S2 셀 또는 100~300m 반경 흐림으로 낮춘다. 민감 장소는 더 넓게 흐린다.

## 본인확인 기반 쓰기

공개 읽기는 로그인 없이 가능해야 한다. 제보, 현장 판단, 반론, 정정, 권리침해 신고, 알림 설정, `/me/*` 조회는 국내 본인확인 완료 세션이 필요하다. v1 provider는 포트원이며, 별도 회원가입 화면을 만들지 않고 본인확인 완료 시 자동으로 사용자 세션을 만든다. CI/DI, 이름, 전화번호, 생년월일 원문은 공개 응답에 노출하지 않고, CI/DI는 중복 계정 방지를 위한 HMAC 해시로만 저장한다.

## 알림/후원/신고 금지 원칙

알림은 의미 있는 상태 변화에만 보낸다. 다음은 단독 알림 트리거가 아니다.

- raw report volume 증가
- 신고 수 증가
- 후원 증가
- 조회수/공유 수 증가
- 단일 출처 주장
- 언론 보도량 증가
- 신규 계정 급증
- 동일 문구 반복

후원은 운영비 보전 기능이다. 후원 데이터는 랭킹, 알림, 신뢰도, 지도 노출 계산에 넣지 않는다.

국내 v1에서 후원은 `무슨일 운영 후원`으로만 다룬다. 개인 계좌 공개, 해외 결제, 세액공제 기부금 영수증, 후원자 우대 노출, 후원자 배지는 만들지 않는다. PG 연결 전까지 결제 기능은 비활성으로 둔다.

신고는 자동 삭제 버튼이 아니다. 권리침해 신고도 Claim으로 저장하고, 마스킹/보류/쟁점 표시/반론 병기를 먼저 검토한다.
